# todo-app 要件まとめ

## 概要

Clojure 製のシンプルなスタンドアロン CUI（コマンドライン）TODO アプリ。

---

## 技術仕様

| 項目 | 内容 |
|------|------|
| 言語 | Clojure 1.11.1 |
| ビルドツール | Leiningen |
| バージョン | 1.0.0 |
| エントリポイント | `todo-app.core/-main` |
| ライセンス | MIT |

---

## 起動方法

### 1. REPL から `-main` を呼び出す

```bash
lein repl
```

```clojure
(-main "add" "買い物をする")
(-main "list")
(-main "done" "1")
(-main "delete" "2")
```

### 2. `lein run` で直接実行する

```bash
lein run add 買い物をする
lein run list
lein run done 1
lein run delete 2
```

### 3. uberjar をビルドして実行する

```bash
# ビルド
lein uberjar

# 実行（プロジェクトルートから）
java -jar target/uberjar/todo-app-1.0.0-standalone.jar add 買い物をする
java -jar target/uberjar/todo-app-1.0.0-standalone.jar list
```

> **注意**: いずれの方法でも、**コマンドを実行したカレントディレクトリ**を基準に `./log/todo.edn` が参照される。
> 異なるディレクトリから実行すると、別のデータファイルが使われるため注意。

### 起動方法による違い

| 方法 | 用途 | 特徴 |
|------|------|------|
| REPL | 開発・動作確認 | 関数を直接呼べる。状態を保ちながら繰り返し試せる |
| `lein run` | 開発・簡易実行 | ソースから直接実行。Leiningen が必要 |
| uberjar | 配布・本番利用 | Leiningen 不要。JRE があればどこでも動く |

## `-main` の引数と動作モード

`-main` は可変長引数（`& args`）を受け取り、**第1引数の値によって動作が切り替わる**。

```
-main の第1引数
  ├── "add"    → タスク追加モード（第2引数以降をタイトルとして使用）
  ├── "list"   → 一覧表示モード
  ├── "done"   → 完了モード（第2引数を ID として使用）
  ├── "delete" → 削除モード（第2引数を ID として使用）
  └── それ以外（nil を含む） → ヘルプ表示モード
```

- **引数なしで呼び出した場合**: 第1引数が `nil` となり、ヘルプが表示される
- **未知のコマンドを渡した場合**: 同様にヘルプが表示される

#### REPL での呼び出し例

```clojure
(-main)              ;; 引数なし → ヘルプ表示
(-main "list")       ;; 一覧表示
(-main "add" "買い物をする")  ;; タスク追加
(-main "done" "1")   ;; ID 1 を完了に
(-main "delete" "2") ;; ID 2 を削除
(-main "foo")        ;; 未知コマンド → ヘルプ表示
```

## データ仕様

### データファイルの位置

- **現在の設定**: 実行カレントディレクトリ直下の `./log/todo.edn`
- **代替候補**（コード内にコメントアウトで記載）: `~/.todo.edn`（ホームディレクトリ固定）

ホームディレクトリ固定にする場合は `core.clj` の `data-file` 定義を変更する:

```clojure
;; 現在の設定（カレントディレクトリ相対）
(def data-file (str "./log/todo.edn"))

;; ホームディレクトリ固定にする場合
(def data-file (str (System/getProperty "user.home") "/.todo.edn"))
```

### データ構造

- **保存形式**: EDN ファイル
- **データ構造**:
  ```edn
  {:next-id 4
   :todos [{:id 1 :title "タスク名" :done false}
           ...]}
  ```
- `next-id`: 次に割り当てる ID（自動インクリメント）
- 各タスクは `:id`（整数）、`:title`（文字列）、`:done`（真偽値）を持つ

## 機能要件

| コマンド | 引数 | 動作 |
|----------|------|------|
| `add <タスク名>` | タスク名（スペース区切り可） | タスクを追加する |
| `list` | なし | タスク一覧を表示する |
| `done <id>` | タスク ID（整数） | タスクを完了状態にする |
| `delete <id>` | タスク ID（整数） | タスクを削除する |
| `help` / 不明コマンド | なし | ヘルプを表示する |

---

## 表示仕様

- タスク一覧の表示形式: `[x]` または `[ ]` + ID（3桁右詰め）+ タイトル
  ```
  [ ]   1. あいうえお
  [x]   2. かきくけこ
  ```
- タスクが空の場合: `タスクはありません。` と表示

---

## エラーハンドリング

- `add` でタスク名が空の場合: エラーメッセージ表示
- `done` / `delete` で ID が数値でない場合: エラーメッセージ表示
- `done` / `delete` で指定 ID のタスクが存在しない場合: エラーメッセージ表示
- EDN ファイルが存在しない・壊れている場合: 初期データ `{:next-id 1 :todos []}` で代替

---

## 制約・備考

- **永続化**: 全操作後に EDN ファイルへ即時保存
- **done 操作は不可逆**: 一度完了にしたタスクを未完了に戻す機能はない
- **uberjar 対応**: `lein uberjar` でスタンドアロン JAR としてビルド可能
